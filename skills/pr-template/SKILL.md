# PR Template Skill

プルリクエストを作成する際に使用するテンプレートを提供します。

## 使用タイミング

- ユーザーがプルリクエストの作成を依頼したとき
- `gh pr create` コマンドを使用してPRを作成する際

## テンプレート構成

以下のテンプレートを使用してプルリクエストを作成してください。

### 基本構造

```markdown
<!-- GitHub Copilot への依頼事項 :コードレビューを実施してコメントをする際は日本語でお願いします。 -->

## issue

<!-- closes #issue番号 と記載することでマージ時に自動でissueがcloseされます -->

## 変更内容

<!-- 行った変更を簡略に記載してください -->

## 確認したこと

<!-- 何を以て変更要件を満たしていると判断したかを記載してください -->

## スクリーンショット

<!--  変更がわかる画面があれば記載してください -->

## 補足事項

<!-- 補足で説明が必要な場合記載してください -->

## PR時のセルフチェック

<!-- PRのセルフチェックをしてください -->

- [ ] 関数や変数の命名は一目で分かるものになってますか？
- [ ] コメントは適切な量で、誰が見ても分かるコメントになってますか？
- [ ] PR内容に関するテストは書きましたか？
- [ ] PR内容に付随する各種ドキュメントは一緒に修正しましたか？
- [ ] IssueをCloseする場合、完了の定義は満たせていますか？
```

## 使用方法

0. **ベースブランチを確認して `--base` を明示する**（リポジトリによってはデフォルトブランチと開発用ブランチが異なる。プロジェクトの運用ルールや直近のマージ履歴から PR の向き先を確認する）。ブランチが未プッシュなら先に `git push -u origin <branch>` する
1. プルリクエスト作成時に、このテンプレートをベースにする
2. **最初の行のGitHub Copilot指示コメント**（`<!-- GitHub Copilot への依頼事項 ... -->`）は**必ず含める**
3. **その他の説明用コメント**は**含めなくて良い**
4. `## issue` セクションには **`closes #番号`** と記載する（マージ時にissueが自動closeされる）
   - 関連issueが存在しない場合は `なし` と記載する
5. 各セクションに適切な内容を記入する

## 実装例

```bash
gh pr create --title "タイトル" --body "$(cat <<'EOF'
<!-- GitHub Copilot への依頼事項 :コードレビューを実施してコメントをする際は日本語でお願いします。 -->
## issue
closes #123

## 変更内容
- 機能Aを追加
- バグBを修正

## 確認したこと
- ユニットテストが全て通過
- 手動テストで動作確認

## スクリーンショット
なし

## 補足事項
なし

## PR時のセルフチェック
- [x] 関数や変数の命名は一目で分かるものになってますか？
- [x] コメントは適切な量で、誰が見ても分かるコメントになってますか？
- [x] PR内容に関するテストは書きましたか？
- [x] PR内容に付随する各種ドキュメントは一緒に修正しましたか？
- [x] IssueをCloseする場合、完了の定義は満たせていますか？
EOF
)"
```
